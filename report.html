01 — Abstract

Project Overview

Health-Tech is a web-based health question and answer platform designed to make medical and health information accessible to everyone in simple, easy-to-understand language. The system provides clear 1–2 line answers to any health-related question, removing the confusion of complex medical terminology.

The platform uses a static database of 100+ pre-loaded health facts for instant responses. It also features voice search and text-to-speech functionality for improved accessibility.

05 — Architecture

System Architecture

1
User Input — User types or speaks a health question on the frontend.
2
Frontend Request — JavaScript sends a POST request to the backend API at /api/ask with the question.
3
Layer 1 — Static Bank — The backend first checks 100+ pre-loaded Q&A pairs using smart keyword matching. If a match is found, it returns instantly.
5
Response — The answer is sent back to the frontend, displayed with a typewriter effect, and read aloud automatically.
06 — Challenges

Challenges Faced

API Key Setup: Configuring environment variables and understanding how .env files work with Node.js was an initial challenge that was resolved by using the dotenv package.

Smart Matching: Simple string matching was not enough to match user questions to the database. A keyword scoring algorithm was built to improve accuracy significantly.

CORS Issues: When the frontend and backend were on different origins, CORS errors appeared. This was fixed by adding the cors middleware in Express.

Responsive 3D: Making the Three.js background work smoothly across all screen sizes and devices required careful pixel ratio and resize handling.

07 — Conclusion

Conclusion & Learning Outcomes

Health-Tech successfully demonstrates a full-stack web application, voice interfaces, and 3D graphics into a practical, user-friendly product. The project covers a wide range of modern web development skills including backend API development, responsive design, and interactive 3D rendering.

Through this project, key concepts like REST APIs, environment configuration, browser APIs, and modern CSS design patterns were learned and applied. The end result is a working, deployable web application that solves a real-world problem — making health information simple and accessible for everyone.
